Ahora vamos a definir el modelo, pero antes vamos a repasar algunos comandos y conceptos muy utiles:

  • Sequential: Eso define una SECUENCIA de capas en la red neuronal
  • Dense: Aade una capa de neuronas
  • Flatten: Recuerdas que las imgenes cmo eran las imagenes cuando las imprimiste para poder verlas? Un cuadrado, Flatten slo toma ese cuadrado y lo convierte en un vector de una dimensin.

Cada capa de neuronas necesita una funcin de activacin. Normalmente se usa la funcin relu en las capas intermedias y softmax en la ultima capa

  • Relu significa que "Si X>0 devuelve X, si no, devuelve 0", as que lo que hace es pasar slo valores 0 o mayores a la siguiente capa de la red.
  • Softmax toma un conjunto de valores, y escoge el ms grande.

  • Durante el entrenamiento, la red tiene que mostrar resultados de loss y accuracy por cada epoch.

### Tu cdigo para la red neuronal de la pregunta 1 aqu ###
# 1. Preparacin de datos
print("Cargando el dataset Fashion MNIST...")
fashion_mnist = tf.keras.datasets.fashion_mnist
(training_images, training_labels), (test_images, test_labels) = fashion_mnist.load_data()

# Normalizacin de imgenes (convertir valores de pxeles de 0-255 a 0-1)
print("\nNormalizando las imgenes...")
training_images = training_images / 255.0
test_images = test_images / 255.0

# 2. Crear el modelo
print("\nCreando el modelo...")
model = tf.keras.models.Sequential([
    # Capa para aplanar la imagen de 28x28 a un vector de 784 elementos
    tf.keras.layers.Flatten(input_shape=(28, 28)),
    # Capa oculta con 128 neuronas y activacin sigmoid
    tf.keras.layers.Dense(128, activation='sigmoid'),
    # Capa de salida con 10 neuronas (una por clase) y softmax
    tf.keras.layers.Dense(10, activation='softmax')
])

# 3. Compilar el modelo
print("\nCompilando el modelo...")
model.compile(
    optimizer='adam',
    loss='sparse_categorical_crossentropy',
    metrics=['accuracy']
)

# 4. Entrenar el modelo
print("\nComenzando el entrenamiento...")
history = model.fit(
    training_images, 
    training_labels, 
    epochs=10,
    batch_size=64,
    validation_data=(test_images, test_labels),
    verbose=1  # Mostrar barra de progreso
)

# 5. Mostrar resultados finales
print("\nResultados finales:")
print(f"Precisin en entrenamiento: {history.history['accuracy'][-1]:.4f}")
print(f"Precisin en validacin: {history.history['val_accuracy'][-1]:.4f}")

Tu respuesta a la pregunta 4.3 aqu:

El aumento en el nmero de neuronas tiene varios efectos importantes:

  • Capacidad del modelo: Al aumentar de 128 a 512 y 1024 neuronas, incrementamos significativamente la capacidad del modelo para aprender patrones ms complejos.
  • Tiempo de entrenamiento: Se observa un aumento considerable en el tiempo de entrenamiento, ya que hay ms parmetros que ajustar.
  • Riesgo de sobreajuste: Con ms neuronas, el modelo tiene mayor riesgo de memorizar los datos de entrenamiento en lugar de generalizar, lo que puede verse en una mayor diferencia entre la precisin de entrenamiento y validacin.
  • Uso de recursos: El consumo de memoria y procesamiento aumenta significativamente.

# Callback mejorado
class myCallback(tf.keras.callbacks.Callback):
    def __init__(self):
        super(myCallback, self).__init__()
        self.loss_history = []
        
    def on_epoch_end(self, epoch, logs={}):
        current_loss = logs.get('loss')
        self.loss_history.append(current_loss)
        
        # Solo detenemos si la prdida es menor a 0.4 despus de algunas epochs
        # para evitar detenciones prematuras
        if epoch > 5 and current_loss < 0.4:
            print(f"\nEpoch {epoch}: La prdida ({current_loss:.4f}) ha bajado del 40%")
            self.model.stop_training = True
